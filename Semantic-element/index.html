    Semantic이란?

    의미론적인, 의미가 통하는

    문서의 정보를 보다 잘 표현하기 위해서는 의미에 맞는 태그를 잘 사용해야합니다.

    HTML5 이전에는 div나 span을 사용했는데, 이것이 어떤 용도로 사용된 것인지 알기가 쉽지 않았고 복잡해지는 단점이 있었습니다.

    이후 HTML5에서는 웹페이지에서 통상적으로 많이 사용하는 구조에 의미를 분명한 부여하기 위해

    Sematic Tag를 새롭게 정의했습니다.


    • header : 웹페이지 상단에 사이트에 대한 전체적인 정보
    • nav : 컨텐츠를 탐색할 때 사용하는 네비게이션에 해당하는 정보
    • section : 문서의 섹션을 의미함
    • article : 본문
    • aside : 광고와 같이 페이지의 내용과는 거리가 먼 내용들
    • footer : 문서나 섹션의 꼬릿말을 의미하며 주로 저자나 저작권, 연락처 정보 등을 포함
