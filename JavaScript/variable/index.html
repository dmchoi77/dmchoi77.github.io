    • 변수 선언 var, let, const

    2022년 01월 05일

    자바스크립트에는 세 가지의 변수 선언 방식이 존재합니다.

    초창기에 사용했던 var와 ES6부터 새롭게 생긴 let과 const가 있습니다.

    var

    var는 한번 선언된 변수를 다시 선언할 수 있습니다.

    var name = "choi";
console.log(name); // choi

var name = "kim";
console.log(name); // kim

    하지만, 이러한 특징은 대규모 프로젝트 진행 시 위험성이 있습니다.

    예를 들어, 같은 프로젝트를 진행하고 있는 A 개발자가 var num을 사용했는데 B 개발자는 이를 모르고 var num을 똑같이 선언해서 사용하게 된다면

    실행할때는 문제가 되지 않기 때문에, 서로 같은 변수명을 선언 했는지 인지하지 못하지만

    이는 나중에 문제가 될 수 있습니다. 그래서 var 선언을 권장하지 않습니다.

    let

    let은 중복 선언이 불가능합니다.

    let num = 3;
console.log(num); // 3

let num = 5; // 중복 선언이 되지 않기 때문에 에러 발생!

    let을 사용하면 똑같은 변수를 선언하는 일이 적어져 안정성이 높아집니다.


    let은 재할당이 가능합니다.

    let name = 'choi';

name = 'kim;';

    const

    const는 let처럼 변수 중복 선언이 불가능하며, 재할당도 불가능합니다.

    const num = 10;

const num = 20; // 에러발생

    스코프(Scope)

    Scope란 우리말로 ‘범위’ 라는 뜻을 가지고 있습니다. 즉 ‘변수에 접근할 수 있는 범위’ 입니다.

    변수마다 다른 스코프를 가지고 있습니다.

    먼저 var는 함수 스코프입니다.

    var로 선언한 변수는 오로지 함수의 코드 블록만을 지역 스코프로 인정합니다.

    함수 외부에서 var로 선언한 변수는 코드 블록 내 어디서든 선언해도 모두 전역 변수가 됩니다.

    var x = 1;
if (true) {
// x는 전역 변수. 이미 선언된 전역 변수 x가 있으므로 x 변수는 중복 선언됩니다.
// 이는 의도치 않게 변수값이 변경되는 부작용을 발생시킵니다.
var x = 10;
}
console.log(x); // 10
    var i = 10;
// for문에서 선언한 i는 전역 변수입니다. 이미 선언된 전역 변수 i가 있으므로 중복 선언이 됩니다.
for (var i = 0; i < 5; i++) {
	console.log(i); // 0 1 2 3 4
}
// 의도치 않게 i 변수의 값이 변경되었습니다.
console.log(i); // 5

    let과 const는 블록 스코프입니다.

    let과 const로 선언한 변수는 함수, if문, for문, while문, try/catch문 등을 지역 스코프로 인정하는 블록 레벨 스코프를 따릅니다.

    아래 코드와 같이 간단한 예시로 확인해 볼 수 있습니다.

    let foo = 1; // 전역 변수

{
  let foo = 2; // 지역 변수
  let bar = 3; // 지역 변수
}

console.log(foo); // 1
console.log(bar); // ReferenceError: bar is not defined
